记录.搞个服务器来干啥？

本文最后更新于：2021年8月21日 下午

说明

本文章主要提供于，当拥有了闲置的服务器的时候，能拿来做什么，操作教程基本通用，因为大多的运行环境都是基于 Linux 。
而且服务器不单单指电脑，腾讯云，阿里云等。手机也可以作为一个服务器，详细可以了解
Termux
和 Linux Deploy 。

我的服务器环境

- - -
服务器 腾讯云服务器 CPU：单核，
内存：1G，
带宽：1M
系统 CentOS 7.6 64位
管理面板 宝塔 简单粗暴的后台管理面板，
如果对 Linux 熟悉，不建议安装面板，这样会省下很多的资源占用。

个人网站

作用

放一些自己的小作品，或者利用 Hexo 搭建个博客。

注意事项

  1. 带宽，如果太小，加载会比较慢。
  2. 端口，例 Hexo，默认显示在端口 4000，而服务器平常的开放的网站访问端口是 80 ，需要反向代理或者修改。

我的操作（已放弃）

服务器的带宽太小（1M），一些文章的图片多的话，加载过慢。
可以将博客搭建在 Github ，
然后将服务器重定向到真实地址，例：https://blog.llxbh.xyz/

Nexus 仓库

作用

用于开发时的依赖加速，可以避免 Github 或者网络罢工的时候，无法下载下来依赖。
而且 Nexus 设置好代理后，会先检查仓库，没有这个文件才远程下载给用户。
搭建可以参考文章：Post not found: 日志-利用_Docker_搭建_Nexus_仓库私服 利用 Docker 搭建 Nexus

注意事项

  1. 管理，Nexus 一般用于内部局域网使用，不适合公开。
  2. 带宽，如果带宽太小，使用的人过多的话，严重影响下载速度。
  3. 代替品，单纯想解决依赖下载慢问题，可以用国内公开仓库，例：阿里 Mavne 镜像

我的操作（已放弃）

带宽太小导致下载依赖回本地的速度过慢，不如直接用阿里云公开仓库。后来使用了闲置手机搞了个服务器，电脑直接在局域网内进行访问😜。

离线/远程 下载服务

作用

  1. 最简单，最直接 -> 用来下载学习资料👍
  2. 稀缺资源下载慢的时候不用一直挂着，可以由服务器下载完了再取回来。

注意事项

  1. 版权，如果使用 BT 下载了侵犯版权的文件，有一定的可能性会被服务商封停。
  2. 带宽，毕竟还是要把东西下载回本地的，如果带宽小，拉取速度慢，不如直接本地下载好。
  3. 适用资源，适用于下载冷门资源，下载速度过慢（几十kb/s）的类型，当然，你带宽大当我没说😂。
  4. 管理，随意公开使用，容易会被别人使用，建议建立一定门槛隔绝或者登录系统。
